企业工作上下文通常分散在 Outlook 邮件与日历、Teams 对话、SharePoint 和 OneDrive 文件中。下面的视频演示了如何在 **Microsoft Foundry Agent** 中添加 Work IQ 工具，让 Agent 根据当前用户的 Microsoft 365 上下文回答问题，并通过 Routine 完成每日重复工作内容。

## 视频中完成了什么？

整个过程可以概括为四步：

1. 在 Foundry Agent 中添加 Work IQ Calendar、Teams 和 Mail 工具。
2. 询问“明天有哪些会议，哪一场需要重点关注”，由 Agent 自动选择 Calendar 工具、提取参数并重写查询。
3. 询问“本周和 Austin 讨论了什么”，由 Agent 组合 Teams 与 Mail 两个数据源生成总结。
4. 使用 Work IQ Copilot 汇总更广泛的 Microsoft 365 上下文，并通过 Foundry Routine 定期运行日报或周报任务。

视频展示了配置和调用过程，下面重点补充这些能力背后的价值与差异。

## Work IQ 的核心价值是什么？

大模型拥有通用知识，但并不知道“我明天有哪些会议”“我和某位同事刚刚讨论了什么”。这些信息实时变化、分散在不同应用中，并且受到企业权限保护。

**Work IQ 是支撑 Microsoft 365 Copilot 和 Agent 的工作智能层。** 它连接邮件、会议、聊天和文件等工作信号，并加入语义理解，使 Agent 不只是找到几条记录，还能围绕人物、项目、时间和任务理解这些记录之间的关系。

在视频的会议场景中，普通日历 API 可以列出会议；结合 Work IQ 后，Agent 还可以根据主题、参会人和可用上下文分析哪些会议更值得关注。在第二个场景中，用户不必分别搜索 Teams 和邮件，只需表达工作目标，Agent 会决定应该使用哪些工具，并将跨数据源结果整理为一个答案。

这意味着 Foundry Agent 与 Work IQ 的分工是：

- **Foundry Agent 负责理解目标和编排工具**；
- **Work IQ 负责提供受权限保护的实时工作上下文**；
- **模型负责归纳、推理并生成最终回答**。

Work IQ 并不是把整个 Microsoft 365 数据复制给模型，也不会绕过现有权限。请求以当前登录用户身份执行，并继续遵守 Microsoft 365 权限和敏感度标签。用户原本无权访问的内容，Agent 也不能通过 Work IQ 获取。

## MCP 工具与 Work IQ Copilot 有什么区别？

视频中展示了两种使用方式，它们解决的问题并不相同：

| 方式 | 适合的场景 | 特点 |
| --- | --- | --- |
| Work IQ Mail、Calendar、Teams 等 MCP 工具 | 查询会议、搜索邮件、读取对话或创建实体等边界清晰的任务 | 操作明确，方便限制权限、测试和审计 |
| Work IQ Copilot | “总结本周工作”“汇总某项目最新进展”等开放式问题 | 可综合 Teams、邮件、文件等多种上下文进行语义推理 |

一个实用的选择原则是：**边界明确、需要确定操作的任务优先使用细粒度工具；数据源不确定、需要跨源理解时使用 Work IQ Copilot。** 工具并非越多越好，只提供完成任务所需的最小工具集，既能降低误调用，也更符合最小权限原则。

## 从回答问题到主动工作：Foundry Routine

视频最后创建了一个定时执行的 Daily Work Report。Foundry Routine 可以在指定时间调用已有 Agent，并保留运行状态、输出及相关 Trace。

它适合“一个触发器调用一个 Agent”的轻量自动化，例如：

- 每天早晨整理会议准备清单；
- 下班前汇总完成事项与未决问题；
- 每周生成项目进展、风险和下一步行动。

这样，Agent 就不再只是等待用户提问，而是开始进入真实的工作节奏。如果任务包含条件分支、多个 Agent 或人工审批，应使用 Workflow 等编排方式。对于发送邮件、修改会议等写操作，也应增加确认或审批，而不是直接无人值守执行。

## 总结

Work IQ 的价值不只是让 Agent “搜索 Microsoft 365”，而是让分散在邮件、会议、聊天和文件中的工作信号，成为 Agent 可以在权限边界内理解和使用的上下文。

结合 Foundry Agent 的模型编排、工具调用、Tracing 与 Routine，我们可以构建一个真正理解个人工作状态的 Agent：它知道去哪里寻找信息，能够跨来源形成总结，解释任务优先级，并在约定的时间主动完成工作。

**You can outsource your thinking, but you cannot outsource your understanding**
